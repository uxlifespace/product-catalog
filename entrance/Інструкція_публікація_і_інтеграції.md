# Як вивести сторінку Entrance в інтернет: домен, кнопки, KeyCRM

Сторінка — статичний `entrance.html` (HTML/CSS/JS, без бекенду). Нижче — повний алгоритм від файлу до робочої воронки.

---

## Частина 1. Публікація сторінки + домен

### Крок 1. Підготувати файл
1. Перейменуй `entrance.html` → `index.html` (хостинг автоматично відкриває саме `index.html`).
2. Переконайся, що всі SVG **вбудовані inline** у файл (вони вже вбудовані) — тоді нічого не загубиться при завантаженні.

### Крок 2. Обрати хостинг (рекомендація — безкоштовний статичний)
Будь-який підійде, але найпростіші для одного файлу:

| Варіант | Кому | Плюси |
|---|---|---|
| **Cloudflare Pages** (рекомендую) | потрібен безкоштовний HTTPS + швидкість | безкоштовно, свій домен, миттєвий деплой |
| **Netlify** | простий drag-and-drop | перетягнув файл — і готово |
| **Vercel** | якщо далі буде serverless-функція (див. Частину 3) | вбудовані функції для KeyCRM |
| Хостинг твого домену (cPanel) | якщо домен уже на платному хостингу | покласти файл у `public_html` |

### Крок 3. Залити сторінку
**На прикладі Netlify (найшвидше):**
1. Зайди на app.netlify.com → **Add new site → Deploy manually**.
2. Перетягни папку з `index.html`.
3. Отримаєш тимчасову адресу типу `random-name.netlify.app` — сторінка вже онлайн.

### Крок 4. Підв'язати свій домен
1. У панелі хостингу: **Domain settings → Add custom domain** → впиши свій домен (напр. `marcelco.com.ua`).
2. Хостинг покаже, які **DNS-записи** додати. Зайди до реєстратора домену (там, де купував) у розділ **DNS**:
   - Для кореня домену: запис типу `A` або `ALIAS/ANAME` на IP/адресу хостингу (хостинг підкаже значення).
   - Для `www`: запис `CNAME` → на адресу хостингу.
3. Збережи. Поширення DNS — від 10 хв до 24 год.
4. Увімкни **HTTPS / SSL** (на Netlify/Cloudflare/Vercel — автоматично, кнопка «Provision certificate»).

✅ Після цього сторінка відкривається за твоїм доменом із замочком (https).

## Частина 2. Кнопки → зовнішні посилання

У файлі кнопки вже є тегами `<a>`. Треба лише замінити заглушки `href="#..."` на реальні адреси.

1. Відкрий `index.html`, знайди три кнопки:

```html
<a class="option" id="btn-catalog" href="#catalog" ...>      <!-- Каталог -->
<a class="option" id="btn-instagram" href="#instagram" ...>   <!-- Instagram -->
<a class="option" id="btn-telegram" href="#telegram" ...>     <!-- Telegram -->
```

2. Заміни `href`:

```html
<a ... id="btn-catalog"   href="https://ТВІЙ-КАТАЛОГ-АБО-САЙТ">
<a ... id="btn-instagram" href="https://www.instagram.com/ТВІЙ_АКАУНТ/" target="_blank" rel="noopener">
<a ... id="btn-telegram"  href="https://t.me/ТВІЙ_БОТ" target="_blank" rel="noopener">
```

- `target="_blank"` — відкривати в новій вкладці (добре для Instagram/Telegram, щоб не втрачати лендинг).
- Каталог — якщо це окрема сторінка того ж сайту, `target` можна не ставити.

3. Збережи й перезалий файл на хостинг (на Netlify — знову перетягнути, або підключити Git для авто-деплою).

## Частина 3. Номер телефону → воронка в KeyCRM

KeyCRM має відкритий REST API (`https://openapi.keycrm.app/v1`), де заявка створюється запитом до **воронки** (`pipelines/cards`). Авторизація — через **API-ключ** (Налаштування → розділ загальних / API).

⚠️ **Головне правило безпеки:** API-ключ дає доступ до всієї CRM. Його **не можна** писати у `index.html` чи в JS на сторінці — інакше будь-хто побачить його в коді й отримає доступ. Потрібна проміжна ланка, яка зберігає ключ у себе. Є 2 шляхи.

### Варіант A (без коду) — через сервіс-конектор
Найшвидше, якщо не хочеш писати бекенд.
1. Зареєструйся в **SaveMyLeads** або **ApiX-Drive** (підтримують KeyCRM офіційно).
2. Створи зв'язку: **Джерело = Webhook (форма)** → **Отримувач = KeyCRM → Create Lead**.
3. Сервіс дасть тобі **URL вебхука**. Вставляєш цей URL у форму сторінки (поле `action`).
4. У налаштуваннях отримувача вкажи свій **API-ключ KeyCRM** і **воронку/статус**, куди падатиме заявка. Зістав поле `phone` → телефон контакту.
5. Тест: відправ номер із сторінки → перевір, що картка з'явилась у воронці.

### Варіант B (свій код) — serverless-функція (рекомендую для контролю)
Якщо хостинг — **Vercel** або **Netlify**, можна додати безкоштовну функцію-посередник.

1. Створи файл `api/lead.js` (Vercel) поряд зі сторінкою:

```js
// api/lead.js  — приймає номер зі сторінки і створює заявку в KeyCRM
export default async function handler(req, res) {
  if (req.method !== 'POST') return res.status(405).end();
  const { phone } = req.body || {};
  if (!phone) return res.status(400).json({ error: 'no phone' });

  const r = await fetch('https://openapi.keycrm.app/v1/pipelines/cards', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.KEYCRM_API_KEY}`, // ключ — у змінних оточення, не в коді
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'Заявка з лендингу Entrance',
      pipeline_id: 1,        // ID твоєї воронки — подивись у KeyCRM / Swagger
      source_id: 1,          // джерело (опційно)
      contact: { full_name: 'Заявка з сайту', phone }
    })
  });
  const data = await r.json();
  return res.status(r.ok ? 200 : 502).json(data);
}
```

2. У налаштуваннях хостингу додай **Environment Variable**: `KEYCRM_API_KEY = твій_ключ`. Так ключ лишається на сервері.
3. У `index.html` у JS-обробнику форми (там, де зараз `// Hook up real submission here.`) додай відправку:

```js
fetch('/api/lead', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ phone: input.value })
});
```

4. Точні назви полів (`pipeline_id`, `source_id`, структура `contact`) звір у Swagger: **https://docs.keycrm.app/** — там видно реальні параметри `POST /pipelines/cards` для твого акаунта.

### Крок перевірки воронки в KeyCRM
1. У KeyCRM: **Воронки → створи воронку** (напр. «Заявки з лендингу») і статуси (Нова → В роботі → Закрита).
2. Запам'ятай **ID воронки** — він потрібен у коді/конекторі.
3. Зроби тестову заявку зі сторінки → картка має з'явитись у статусі «Нова».

## Підсумковий чек-лист
1. ☐ `entrance.html` → `index.html`.
2. ☐ Залити на Netlify/Cloudflare/Vercel.
3. ☐ Підв'язати домен через DNS (A/CNAME) + увімкнути HTTPS.
4. ☐ Прописати реальні `href` для Каталогу, Instagram, Telegram.
5. ☐ Створити воронку в KeyCRM + взяти API-ключ та ID воронки.
6. ☐ Налаштувати міст: SaveMyLeads/ApiX-Drive (без коду) **або** serverless-функцію (свій код).
7. ☐ Підключити форму до моста й протестувати наскрізно.
8. ☐ Перевірити, що заявка падає у воронку.
